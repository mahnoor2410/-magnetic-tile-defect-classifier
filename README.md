# Magnetic Tile Surface Defect Classifier — ML Engineering Assessment

End-to-end **computer vision** prototype for magnetic tile surface images: **data preparation**, **two-model training**, **evaluation & error analysis**, and **inference (script + REST API)**. Built for the AI/ML Engineer Technical Skills Assessment (Computer Vision Focus).

Python **3.9+** recommended.

---

## What this solves

- **Task:** Multi-class image classification — 6 classes: `Blowhole`, `Break`, `Crack`, `Fray`, `Uneven` (defects) + `Free` (no defect).
- **Dataset:** [Magnetic Tile Surface Defects](https://www.kaggle.com/datasets/alex000kim/magnetic-tile-surface-defects) (Kaggle) — ~1,344 images, meaningfully imbalanced, near-grayscale JPEGs with inconsistent resolutions. Chosen over Severstal (too large for the 3-day window) and NEU (too clean/balanced to demonstrate imbalance handling).
- **Split:** Stratified 70/15/15 train/val/test (fixed seed, preserves class proportions given the imbalance).
- **Why classification, not segmentation:** the assessment flags segmentation as optional/higher-difficulty — a thoroughly-executed classification pipeline scores better within 3 days than a shallow segmentation attempt.

---

## Models used — and why

| Model | Role | Why |
|---|---|---|
| **Baseline CNN** (`BaselineCNN`) | Required "simple baseline" | 4 conv blocks, trained from scratch. Quantifies the actual lift transfer learning provides — a reference point, not an assumption. Fast on CPU. |
| **ResNet18** (fine-tuned) | Required "stronger" model | ImageNet-pretrained. Staged fine-tune: 5 epochs with backbone frozen (head only), then `layer4` + head unfrozen at low LR. ResNet18 (not ResNet50) specifically for compute-awareness — fast CPU inference, less overfitting risk on ~1,300 images. |

**Imbalance handling:** inverse-frequency **class-weighted loss**, not oversampling — with this little data, oversampling risks the model memorizing duplicated minority-class images.

**Metrics:** **macro-F1** + per-class precision/recall alongside accuracy, since a model always predicting `Free` would score high accuracy while being useless for defect detection.

## Repository layout (`src/` separation)

| Path | Responsibility |
|---|---|
| `src/data/{dataset,transforms,analysis}.py` | Deliverable 1 — scanning, stratified split, transforms/augmentation, EDA figures |
| `src/models/{baseline_cnn,resnet_finetune}.py` | Model architectures |
| `src/training/{train,utils}.py` | Deliverable 2 — training loop, class weights, checkpointing on best val macro-F1 |
| `src/evaluation/{evaluate,error_analysis}.py` | Deliverable 3 — test-set metrics, confusion matrix, qualitative error analysis |
| `src/inference/{predictor,api}.py` | Deliverable 4 — script inference (`DefectPredictor`) + FastAPI REST inference |
| `config/config.yaml` | All paths/hyperparameters — single source of truth, loaded via `src/config_loader.py` |

**Artifacts:** under `outputs/{checkpoints,figures,metrics,logs}/` — generated by the scripts, safe to regenerate.

## Environment setup

```bash
python -m venv venv
source venv/bin/activate        # Windows: venv\Scripts\activate
pip install -r requirements.txt
```

Place the dataset under `data/raw/` so it looks like:
```
data/raw/MT_Blowhole/Imgs/*.jpg   (also MT_Break, MT_Crack, MT_Fray, MT_Uneven, MT_Free)
```

---

## How to run (CLI)

### Deliverable 1 — Data preparation

```bash
python -m src.data.analysis
```
Outputs class distribution + sample grid to `outputs/figures/`.

### Deliverable 2 — Training

```bash
python -m src.training.train --model baseline
python -m src.training.train --model resnet
```
Checkpoints saved to `outputs/checkpoints/{baseline,resnet}_best.pth` on every val macro-F1 improvement. Baseline: 25 epochs. ResNet: 15 epochs (5 frozen + 10 partial fine-tune).

### Deliverable 3 — Evaluation

```bash
python -m src.evaluation.evaluate --model baseline
python -m src.evaluation.evaluate --model resnet
python -m src.evaluation.error_analysis --model resnet
```
Outputs metrics JSON + confusion matrix to `outputs/metrics/` and `outputs/figures/`.

### Deliverable 4 — Inference (script)

```python
from src.inference.predictor import DefectPredictor
predictor = DefectPredictor(checkpoint_path="outputs/checkpoints/resnet_best.pth", model_type="resnet", device="cpu")
print(predictor.predict("data/raw/MT_Crack/Imgs/exp1_num_10.jpg"))
```

### Deliverable 4 — Inference (FastAPI)

```bash
uvicorn src.inference.api:app --reload --host 0.0.0.0 --port 8000
```
`POST /predict` (multipart file upload), `GET /health`, `GET /classes`.
```bash
curl -X POST "http://localhost:8000/predict" -F "file=@data/raw/MT_Crack/Imgs/exp1_num_10.jpg;type=image/jpeg"
```
Interactive docs: `http://localhost:8000/docs`.

### Tests

```bash
pytest tests/ -v
```
Runs offline with a synthetic checkpoint — no dataset or trained model required.

## Results

| | Accuracy | Macro-F1 |
|---|---|---|
| Baseline CNN | 47.0% | 0.173 |
| **ResNet18** | **85.6%** | **0.768** |

Transfer learning gives a large lift over the from-scratch baseline, which struggles on minority classes (`Blowhole`, `Fray` near-zero recall).

**Error analysis** (`outputs/metrics/resnet_error_analysis.json`, `outputs/figures/resnet_misclassified_samples.png`): dominant failure mode is **`Free ↔ Break` confusion** (11 Free→Break, 7 Break→Free of 29 total errors) — subtle break-like textures on defect-free tiles appear to be the cause. Suggests higher-resolution crops or hard-negative mining on this specific pair as a concrete next step, rather than blind hyperparameter tuning.

**If more compute were available:** ResNet50/EfficientNet-B0, proper LR/weight-decay grid search, synthetic defect augmentation for minority classes, k-fold cross-validation.
